| 状態 | 意味 |

| 項目 | 状況 |
| --- | --- |

## 次に開発が必要な項目

### 優先度 高

| 項目 | 理由 |
| --- | --- |
| チームノートのGAS反映と再デプロイ | ローカル実装だけでは実際に使えないため |
| `TeamMember` / プロフィール構造の整理 | 今後のアクセス制御と一覧表示の土台になるため |
| Student / General の閲覧制御強化 | 設計思想上、最重要の安全要件のため |
| 部員一覧UI | 現場運用での実用性が高いため |

### 優先度 中

| 項目 | 理由 |
| --- | --- |
| 個人ページ | 振り返りの蓄積を見返しやすくするため |
| 検索機能 | 人数が増えた時に必須になるため |
| 指導内容 / クレド機能 | 指導文化の言語化と継承に直結するため |
| ログイン履歴の実運用 | 管理面・保守面で必要になるため |

### 優先度 低

| 項目 | 理由 |
| --- | --- |
| AI分析 | データ蓄積が先のため |
| グローバル共有 | Student/General 制御が固まってからの方が安全なため |
| 指導者課金機能 | まだプロダクト基盤優先の段階のため |

## 現時点の総評

現状のKEIKO™︎は、設計図全体の中では「MVPの前半」がかなり形になってきた段階です。  
特に、`ログイン/登録/所属チーム/ID移行/稽古ログ/Good&New/ライブラリ` までは基盤ができています。

一方で、設計図の本丸である

- 厳密なアクセス制御
- 個人ページ
- 部員一覧UI
- 指導内容の蓄積
- AI分析
- グローバル共有

はまだこれからです。

また、今回追加した `チームノート` は、事業構想の「個人の振り返り」から一歩進んで「指導内容に対する学びの共有」を実現する重要機能であり、次の中核機能になりうる状態です。  
ただし、これはまだ **ローカル実装済み / GAS反映待ち** のため、運用に入れるには再デプロイが必要です。
